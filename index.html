es6-prototype

ES6中的原型增强

  • 新增方法Object.setPrototypeOf(obj, obj1)方法
    该方法可以改变任意指定对象的原型，第一个参数为被改变原型的对象，第二个参数为替代对象

  • 新增super方法
    super方法引用原型对象上的方法，且引用不是动态化的,它总是指向正确的对象

  • ES6中每个方法内部都有一个[[HomeObject]]属性来容纳这个方法的从属对象
    super的所有引用都通过[[HomeObject]]属性来确定后续的运行过程。

let grandFather = {
    getGreeting() {
        return 'hello';
    }
};

let father = {
    getGreeting() {
        return super.getGreeting() + ", hi!";
    }
}

Object.setPrototypeOf(father, grandFather);

let child = Object.create(father);
console.log(grandFather.getGreeting());//'hello'
console.log(father.getGreeting());//'hello, hi!'
console.log(child.getGreeting());//'hello, hi!'

child的原型为father,father的原型为grandFather;
我们可以在对象被创建后修改原型，使用super关键字调用原型上的方法;

extend

重要概念

继承

创建一个派生类，并使其复用基本类（即父类）的字段和／或方法。并且派生类可以重写基本类的方法。这样基本类和派生类相同签名的方法在被调用时，就会有不同的行为表现，即为多态的实质。换句话说，多态是透过继承和重写实现的。

ES5中的继承

  • 原型链继承
    就是通过把子类的prototype指向父类的实例

  • 构造函数继承
    就是通过在子类内部调用call/apply方法继承父类方法属性

  • 组合继承
    就是通过构造函数定义实例属性，原型继承父类的共享属性和方法

  • 寄生组合继承
    就是通过借用构造函数继承属性，通过原型链的混合形式继承方法

currying

重要概念

柯里化

  • 把接受多个参数的函数变换成接受一个单一参数（最初函数的第一个参数）的函数，并且返回接受余下的参数而且返回结果的新函数的技术。
  • 柯里化通常也称部分求值，其含义是给函数分步传递参数，每次传递参数后部分应用参数，并返回一个更具体的函数接受剩下的参数，这中间可嵌套多层这样的接受部分参数函数，直至返回最后结果。

原理

方法一传递一个参数，返回方法二，方法二递归调用，当参数为空时返回结果。
期间方法一的私有变量会产生闭包，保存了方法二的结果。

通用的函数来对普通函数进行柯里化

ES6下的柯里化

inject

重要概念

  • 控制反转(Inversion of Control)
    把创建对象等操作，交给IOC容器控制，并不是在当前内部对象中直接控制。
    比如注入一个服务（object），并不是直接new一个服务，而是告诉框架要注入一个服务，等待框架返回已经实例化好的对象。
    对对象的控制权反转了，从当前对象中反转到了IOC容器

  • 依赖注入(DI)
    动态的向某个对象提供它所需要的其他对象

优点

  1. 解耦，降低组件之间的耦合度
  2. 可以使用容器提供的众多服务，如：事务管理服务、消息服务等等。当我们使用容器管理事务时，开发人员就不再需要手工控制事务.也不需处理复杂的事务传播
  3. 容器可提供单例支持

浏览器缓存

重要概念

  • 强缓存
    用户发送的请求，直接从客户端缓存中获取，不发送请求到服务器，不与服务器发生交互行为。
    状态码 200（from cache）

  • 协商缓存
    用户发送的请求，发送到服务器后，由服务器判定是否从缓存中获取资源。
    状态码 304（not modified）

response header缓存相关参数

强缓存相关

  1. expires
    Http1.0 中的标准，表明过期时间，注意此处的时间都是指的是服务器的时间。
    如果发送请求的时间在expires之前，那么本地缓存始终有效，否则就会发送请求到服务器来获取资源。

  2. cache-control
    缓存策略，是一个相对值
    2.1 max-age: 设置缓存的最大的有效时间，单位为秒（s）。max-age会覆盖掉Expires
    2.2 s-maxage: 只用于共享缓存，比如CDN缓存（s -> share）。与max-age 的区别是：max-age用于普通缓存，
    而s-maxage用于代理缓存。如果存在s-maxage,则会覆盖max-age 和 Expires.
    2.3 public：可以被所有的用户缓存，包括终端用户和CDN等中间代理服务器。默认是public。
    2.4 private: 只能被终端用户的浏览器缓存，不允许CDN等中继缓存服务器对其缓存。
    2.5 no-cache: 不使用本地缓存。但是设置了no-cache之后并不代表浏览器不缓存，而是在缓存前要向服务器确认资源是否被更改（协商缓存）。如果之前的响应中存在ETag，那么请求的时候会与服务端验证，如果资源未被更改，则可以避免重新下载。
    2.6 no-store: 绝对禁止缓存，每次用户请求该资源，都会向服务器发送一个请求，每次都会下载完整的资源。

协商缓存相关

  1. Last-modified
    表明请求的资源上次的修改时间。在respone header上
  2. If-Modified-Since
    客户端保留的资源上次的修改时间。在request header上，等于Last-modified
  3. Etag
    资源的内容标识，只要资源有变化就这个值就会改变
  4. If-None-Match
    客户端保留的资源内容标识，只要资源有变化就这个值就会改变

使用浏览器缓存判断过程

首次加载页面

  1. 浏览器发出请求
  2. 判断浏览器本地是否有缓存–无缓存
  3. 向web服务器发送请求
  4. 请求响应并缓存资源（缓存相关response header）

再次加载页面–本地有缓存时

强缓存 (request header)

  1. 查看是否有cache-control 的max-age / s-maxage , 如果有，则用服务器时间date值 + max-age/s-maxage 的秒数计算出新的过期时间，将当前时间与过期时间进行比较，判断是否过期。
    如果有no-cache,则请求服务器（协商缓存）。
    如果有no-store,则请求服务器（不执行协商缓存）。
  2. 如果没有cache-control 则用expires 作为过期时间比较

协商缓存 (没有发生强缓存时)

  1. 服务器接受到请求，先根据Etag,判断资源是否变化。
    如果相同则判断Last-Modified。
    如果不同则直接返回新的数据并刷新header。
  2. 根据浏览器传过来If-Modified-Since和资源在服务器上的最后修改时间判断资源是否有变化。
    如果没有变化则返回304 Not Modified，但是不会返回资源内容。
    如果有变化，就正常返回资源内容。当服务器返回304 Not Modified的响应时，response header中不会再添加Last-Modified的header。
